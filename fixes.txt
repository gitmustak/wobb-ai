Bug Fixes, Quality Improvements & Feature Changes
==================================================

--- Bug Fixes ---

1. Engagement rate calculation (ProfileDetailPage.tsx)
   - Bug: engagement_rate was multiplied by 10000 instead of 100, inflating displayed values 100x (e.g., showing 142.50% instead of 1.43%)
   - Fix: changed multiplier from 10000 to 100

2. Engagements stat card showing wrong value (ProfileDetailPage.tsx)
   - Bug: the "Engagements" card label showed the engagement rate percentage via formatEngagementRate() instead of the actual engagement count
   - Fix: replaced formatEngagementRate(user.engagement_rate) with formatFollowersDetail(user.engagements)

3. Dead clickCount state and ignored username argument (SearchPage.tsx)
   - Bug: handleProfileClick incremented a clickCount state that was never rendered or used, and silently ignored the username argument passed to it
   - Fix: removed clickCount state and the handleProfileClick function entirely; ProfileCard handles navigation independently

4. Unhandled promise rejection causing infinite loading spinner (ProfileDetailPage.tsx)
   - Bug: loadProfileByUsername().then(...) had no .catch(), so any thrown error would leave loaded as false and show the spinner forever
   - Fix: added .catch() that sets profileData to null and loaded to true, showing the "could not load" error state instead

5. Missing useEffect cleanup causing stale state updates (ProfileDetailPage.tsx)
   - Bug: if the user navigated away before the profile loaded, setProfileData/setLoaded would be called on an unmounted component
   - Fix: added a cancelled flag returned as cleanup from useEffect; state setters are no-ops if cancelled is true

6. Unused SearchBar component (SearchBar.tsx)
   - Bug: SearchBar.tsx was never imported or used anywhere; PlatformFilter contains its own inline search input
   - Fix: deleted the file

7. Hardcoded fixed width on ProfileCard (ProfileCard.tsx)
   - Bug: w-[700px] caused cards to overflow on screens narrower than ~720px with no responsive fallback
   - Fix: replaced w-[700px] with w-full

8. Missing alt attributes on img tags (ProfileCard.tsx, ProfileDetailPage.tsx)
   - Bug: both profile image tags lacked alt attributes, failing accessibility standards
   - Fix: added alt={profile.fullname} and alt={user.fullname} respectively

9. filterProfiles ignoring the handle field (dataHelpers.ts)
   - Bug: search only matched against username and fullname; YouTube profiles use a separate handle field (e.g., "MrBeast") that was never checked
   - Fix: added handle to the filter check using p.handle?.toLowerCase().includes(normalizedQuery)

10. Duplicate TODO comment (ProfileDetailPage.tsx)
    - Bug: the same TODO comment for "Add to List" appeared on two consecutive lines
    - Fix: removed the duplicate line

--- Performance Optimizations ---

27. Memoized profile derivation in SearchPage (SearchPage.tsx)
    - extractProfiles(platform) and filterProfiles(allProfiles, searchQuery) ran on every render, including renders triggered by unrelated causes (e.g. Zustand store updates from list changes)
    - Wrapped both in useMemo with their actual dependencies ([platform] and [allProfiles, searchQuery]); filtering now only re-runs when the query or platform actually changes

28. Granular Zustand selectors in ProfileCard (ProfileCard.tsx)
    - Before: const { add, remove, contains } = useListStore() subscribed every card to the entire store; adding one profile caused all 10 cards to re-render
    - After: three separate selectors — s => s.add and s => s.remove always return stable function references (never trigger re-renders); s => s.list.some(p => p.user_id === profile.user_id) returns a boolean and only triggers a re-render when this specific card's membership status changes

29. React.memo on ProfileCard (ProfileCard.tsx)
    - Wrapped the component in React.memo so it skips re-renders when the parent ProfileList re-renders but the card's profile and platform props have not changed (e.g. typing in the search box while filtered results stay the same)

30. React.memo on ListRow + stable onRemove reference (SelectedList.tsx)
    - Before: onRemove={() => remove(profile.user_id)} created a new function reference on every render, making memo useless
    - After: onRemove is now the store's remove function directly (a stable reference); ListRow calls onRemove(profile.user_id) internally; with memo, existing rows no longer re-render when a new profile is added to the list
    - SelectedList itself now uses granular selectors (s => s.list, s => s.remove, s => s.clear) instead of subscribing to the whole store

31. Lazy image loading on all profile pictures (ProfileCard.tsx, SelectedList.tsx, ProfileDetailPage.tsx)
    - Added loading="lazy" to every img tag; without this, all external profile picture URLs were fetched simultaneously on mount, competing for network bandwidth
    - Browser now fetches images only as they scroll into view

--- Animated Background ---

37. Added animated background blobs (index.css, Layout.tsx)
    - Three large blurred blobs (480 px / 420 px / 360 px, blur 88 px) positioned at top-left, bottom-right, and bottom-centre of the layout card
    - Three independent CSS keyframe animations (blob-drift-1/2/3) slowly translate and scale each blob; infinite alternate easing gives smooth back-and-forth motion with no jump at the loop point
    - Colours: warm amber (#c8922a), terracotta (#b87060), sage (#7a9e78) — matching the existing palette
    - Opacities are 7–9% in light mode and 9–12% in dark mode so they read as ambient glow without competing with content
    - Blob wrapper uses aria-hidden, pointer-events-none, and -z-10 inside an isolate parent so blobs stay behind all UI and are ignored by assistive technology
